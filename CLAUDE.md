# CLAUDE.md

このリポジトリで作業する際のガイダンス。

## プロジェクト概要

**Skipferry** — フォルダのコピー/移動を行う Python (Tkinter) GUI ツール。単一ファイル構成。
名称は skip（破損ファイルを飛ばす）+ ferry（健全なファイルを渡し船で運ぶ）の造語。
**主目的は破損ファイル対策**: アプリやOSを巻き込んで固まるタイプの破損ファイルがあっても、
処理順で先頭 N 件をスキップして健全なファイルだけを運び切ること。
加えてサブフォルダを含む再帰処理、タイムスタンプ維持、ベリファイ、ごみ箱送り、
無視リストなどを備える。

（リポジトリのフォルダ名は歴史的経緯で `putoff-file-move` のまま。ツール名は Skipferry。）

## 実行

```
python skipferry.py
```

- 追加パッケージが無くても起動・動作する（外部ライブラリはすべて任意）。
- ごみ箱送りは `send2trash` があれば使用し、無ければ Windows の `SHFileOperation`(ctypes)
  へ自動フォールバックする。`send2trash` は任意（`pip install Send2Trash`）。
- ドラッグ&ドロップは `tkinterdnd2` があれば有効（`pip install tkinterdnd2`）。無い場合は
  D&D 無効で通常起動する（欄下に導入案内を表示）。
- 主対象は Windows。ごみ箱送りフォールバックは Windows 専用。

## 構成

- `skipferry.py` — 全ロジックとGUIを含む単一ファイル。
- `CLAUDE.md` — 本ファイル。

## アーキテクチャ

- **国際化 (i18n)** — UI 文言・ログ・エラーメッセージはすべてモジュール先頭の
  `TRANSLATIONS`(`ja`/`en`) と `t(key, lang=None, **kwargs)` を経由する。**新規文言は必ず
  両言語のキーを追加し、リテラルを直接埋め込まないこと**（`t()` は欠落キーを日本語→キーの順で
  フォールバックする）。現在の言語は module-global `CURRENT_LANG`（既定 `ja`）。GUI 上部の
  ラジオボタンで切替え、`App._on_lang_change` が全ウィジェットを破棄して `_build_ui` で
  再構築する（Tk 変数は `_init_vars` で一度だけ生成し保持、無視リスト/ログ Text の内容は
  退避して復元）。**ワーカーは開始時の言語を `cfg["lang"]` で固定**し `self.tr()` で翻訳するため、
  実行中に UI 言語を切替えてもログ言語はぶれない（実行中は言語切替を無効化）。
  モジュール関数 `plan_operation`/`check_dest_path_valid` は `cfg["lang"]`/`lang` 引数で言語を受ける。
- **処理計画 (`plan_operation`, モジュール関数)** — 設定からコピー先ルート・処理対象
  ファイル一覧(無視/先頭スキップ適用後・ソート済)・エラーを算出する。
  **ワーカー本処理と実行前プレビューの両方がこの関数を共有する**（結果がずれないため）。
  コピー先パスの妥当性検査 `check_dest_path_valid`（Windowsで使用不可の文字
  `< > : " | ? *`・制御文字・予約名 CON/PRN/…・末尾の空白/ドット）もここで行い、
  問題があれば `error` に載せる。ドライブの `:` と区切りは splitdrive で除外して構成要素を見る。
- **GUI ドラッグ&ドロップ** — `tkinterdnd2` があれば基底クラスを `TkinterDnD.Tk` に切替え
  （`_BaseTk`）、コピー元/先の Entry に `DND_FILES` を登録。`_on_drop` は `tk.splitlist` で
  波括弧括りのパスを分解し、ファイルが落ちた場合はその親フォルダを採用する。
- **GUI (`App`, tk.Tk)** — パス指定・オプション・無視リスト・進捗・ログを構築。
  実処理はワーカースレッドに委譲し、GUIは触らない。実行時はまず `plan_operation` で
  計画を作り、`_show_preview` で処理先ファイル(先頭10件)を提示して確認を取ってから開始する。
- **ワーカー (`CopyMoveWorker`, threading.Thread)** — 実際のコピー/移動/検証/削除を実行。
  一覧作成は `plan_operation` に委譲。結果は `queue.Queue` へ
  `("log"|"progress"|"ignore_add"|"done", payload)` 形式で送る。
- **リード タイムアウト (`read_timeout_s`, 子プロセス+kill, 無進捗検知)** — 読み取りで固まる
  破損ファイル対策の主目的をより直接的に満たす機能。`cfg["read_timeout_s"]`（秒, 0=無効）が
  正の時のみ、コピーをモジュール関数 `_copy_worker_main` を実行する**常駐子プロセス**
  （`multiprocessing`）へ逃がす。判定は**総時間ではなく無進捗**で行う：**コピー先 dst の
  サイズが指定秒間まったく増えなければ hang とみなして プロセスごと kill**する
  （`_copy_with_timeout` が 0.1 秒刻みで `_safe_size(dst)` を監視し、サイズ変化があれば
  タイマーをリセット）。これにより**大容量で時間がかかるだけの健全ファイルは誤って打ち切らない**。
  Python はスレッドを kill できないがプロセスは kill できるため子プロセス方式にしている。
  子プロセスは hang して kill した時のみ再生成する
  （`_ensure_copy_proc`/`_kill_copy_proc`/`_shutdown_copy_proc`）ので通常時のオーバーヘッドは
  起動1回のみ。0 の時は従来どおりスレッド内で `shutil.copy2` を直接呼ぶ（オーバーヘッド無し）。
  待機は 0.1 秒刻みで停止要求にも即応。**タイムアウトは `TimeoutError`
  としてエラー扱い**にして既存の自動無視/エラースキップ経路へ載せる（破損ファイルとして次回
  自動スキップされる＝主目的に合致）。停止要求が待機中に来た場合は内部例外 `_Stopped` を送出し、
  本処理ループが行を閉じて中断する。タイムアウトで途中まで書かれた dst は `_remove_partial` で掃除。
  **ベリファイ（ハッシュ再読込）は copy 成功後＝元が読めた後なのでスレッド側に残す**（子プロセスは
  copy のみ担当）。子プロセスは `run()` の finally で必ず後始末する。**Windows の spawn 対策**として
  GUI は `__main__` ガード下に置き（子で GUI を再起動しない）、frozen ビルド用に `__main__` 冒頭で
  `multiprocessing.freeze_support()` を呼ぶ。
- **ログファイル出力** — `cfg["log_file"]` にパスがあれば `run()` が追記(`a`)モードで開き、
  `self.log()` が GUI へ送るのと同時に時刻付きでファイルへも書く(逐次 flush)。開始/終了に
  区切り行を出す。書き込み失敗は処理本体を止めない。GUI 側は `_resolve_log_file` で
  パスを決定（空欄ならプログラム（`_config_path` = ini と同じフォルダ）に
  `skipferry_log_<日時>.txt` を自動生成）。
  加えて現在のログ欄を任意保存する `_save_log`（実行不要でいつでも可）も持つ。
- **設定の永続化 (ini)** — `_start` で実行確定時（プレビュー承認後）と、ウィンドウを閉じる時
  （`_on_close` / `WM_DELETE_WINDOW`）に `_save_settings` が現在の設定を
  `skipferry.ini` へ書き出す。次回起動時は `__init__` が `_load_settings` で復元する。
  設定ファイルは**スクリプト（フリーズ時は実行ファイル）と同じフォルダ**（`_config_path`）。
  `configparser`（`interpolation=None`）を使用し `[General]`（各スカラー設定）と `[Ignore]`
  （無視リストを出現順に添字キーで格納。空行/コメント/`%` も保持）の 2 セクション構成。
  **言語 (`CURRENT_LANG`) は UI 構築より前に確定させる必要がある**ため、`_load_settings` を
  `_build_ui` の前に呼ぶ。無視リストはこの時点で Text 未生成のため `_loaded_ignore` に退避し、
  build 後に `_apply_loaded_ignore` で流し込む。ini が無い/壊れている場合は既定値で起動する
  （保存・読込の失敗はいずれも動作を妨げない）。
- **設定内容の書き出し** — `_run` の冒頭で `_log_settings` がオプション設定
  （コピー元/先・サブフォルダ作成・動作/移動時の元処理・ベリファイ・エラースキップ・
  自動無視・先頭スキップ件数・ウェイト・無視パターン数・ログファイル）をログへ出力する。
  無視パターン数はコメント行(`#`)・空行を除外して数える。GUI ログ/ログファイルの両方に出る。
- **ファイル処理ログの前後（同一行方式）** — 各ファイルはコピー**直前**に
  `[done/total] <rel>`(`log_item`) を**改行せず**(`newline=False`)出し、以降の結果は
  `log_append`(時刻プレフィックス無し)で**同じ行の末尾へ追記**する。成功時はコピー確定後に
  ` [コピー完了]`(`log_item_copied`) を追記。移動時はさらに元削除を**別記録として分ける**：
  ごみ箱なら ` [元ファイルをゴミ箱へ]`(`log_item_recycled`)、完全削除なら
  ` [元ファイル削除]`(`log_item_removed`)。エラー時は同じ行へ ` [エラー] <e>`(`log_item_err`)
  を追記して行を閉じる（`rel` は行頭に既出のため重複させない）。自動無視登録はその後の別行。
  結果として1ファイル=1行（例: `[1/2] a.txt [コピー完了] [元ファイルをゴミ箱へ]`）。
  **処理前ログ（開始行）は維持すること**。改行しないことで、固まってもログ最終行が原因ファイルを
  指し、無視/先頭スキップ対象を特定できるという主目的に直結する。
  ログの改行制御はワーカーの `log(newline=)`/`log_append` と GUI の `_log(..., end=)` が担い、
  キュー payload は `("log", (level, text, end))` の3要素。
- **GUI ↔ ワーカー連携** — `App._poll_queue` が `after(100ms)` でキューをポーリングし
  ログ・進捗バー・無視リスト追加を反映。停止は `worker.stop_flag`(Event)、一時停止は
  `worker.pause_flag`(Event) で協調的に行う。
  **ワーカースレッドから Tkinter ウィジェットを直接操作しないこと**（キュー経由必須）。
- **エラー確認ダイアログ (エラースキップ無効時の 再試行/スキップ/終了)** — `skip_error` が
  無効のときにファイル処理でエラーが出ると、ワーカーはキューへ `("ask_error", (rel, err))` を
  送り `error_event`(Event) で GUI の応答を待つ（`_ask_error_action`, 0.1 秒刻みで停止に応答）。
  GUI 側 `_poll_queue` は `_show_error_dialog` でモーダル Toplevel（再試行/スキップ/終了、
  ×＝終了）を出し、選択を `worker.error_response`＋`error_event.set()` で返す。
  本処理はファイル単位を**再試行ループ**（`while True`）で囲み、`retry`=同じファイルをやり直し
  （`log_item` から出し直す）、`skip`=当該だけ飛ばして継続（エラー計上・自動無視・移動時の
  残フォルダ記録は skip/abort 確定時のみ）、`abort`=中断。**エラースキップ有効時は従来どおり
  ダイアログ無しで即スキップ継続**（内部的に `action="skip"` 相当）。再試行で成功した場合は
  エラー計上も自動無視もしない。**再試行/スキップの確定処理はループ内の同一分岐に集約**し、
  失敗のたびに二重計上しないこと。
- **ドライブ接続消失の検知 (エラースキップ有効時)** — ネットワーク/外付けドライブが処理中に
  切断されると全ファイルが延々エラーになるため、**`skip_error` 有効時のみ**、スキップした
  エラーが連続 `DRIVE_LOST_ERROR_THRESHOLD`(モジュール定数, 既定 10) 件に達したら
  コピー元ドライブのルート（例 `D:\`）に `os.listdir` でアクセスできるか確認する
  （`_drive_root` でルート算出→`_drive_accessible` で疎通確認）。取得できなければ接続消失と
  みなし `log_drive_lost` を出して中断（`aborted=True`）、取得できればカウンタをリセットして
  継続（確認は10件ごとに一度で `listdir` の連打を避ける）。連続エラーカウンタ
  `consecutive_errors` は `_run` のファイル処理ループ内で持ち、**コピー成功でリセット**、
  `action=="skip"` 確定時のみ加算する（エラースキップ無効の確認ダイアログ経路は対象外）。
- **一時停止/再開・ウェイト** — いずれもワーカー内でフラグを見て協調的に待機する。
  一時停止は `pause_flag`。ワーカーは各ファイル処理**前**に `_wait_if_paused` を呼び、
  一時停止中は 0.1 秒刻みで待機する（停止要求が来れば即座に抜ける）。GUI 側の
  「一時停止」ボタンがラベルを「一時停止 ⇄ 再開」でトグルする。**一時停止中でも停止できる
  よう `_stop` は `pause_flag` を解除してから `stop_flag` を立てること**。
  OS/アプリ負荷軽減のウェイトは `cfg["wait_ms"]`(ミリ秒)。各ファイル処理**後**に
  `_throttle_sleep` が挟む。待機は 0.1 秒刻みで停止/一時停止に応答する（長いウェイト中でも
  中断・一時停止が効くようにするため、まとめて `sleep` しないこと）。

## 設計上の重要な決定（変更時は維持すること）

- **移動は「コピー→検証→元削除」で統一** している（`os.rename` は使わない）。
  これにより別ドライブ間移動とベリファイを両立させている。
- **タイムスタンプ維持**: ファイルは `shutil.copy2`。フォルダは全ファイル処理**後**に
  `os.utime` を適用する（先に設定すると子ファイル書き込みで親mtimeが変わるため）。
- **ファイル属性/ACL/副次ストリーム(ADS)は既定では操作しない**。システムデフォルトに従う（要件）。
  例外として**任意オプション `preserve_attrs`**（既定オフ, Windows 専用）を有効にした時のみ、
  コピー後に元の**隠し/システム属性**を `preserve_hidden_system_attrs`（`SetFileAttributesW`）で
  コピー先へ反映する（`shutil.copy2` はこれらを引き継がないため）。読み取り専用は copy2 が
  モードビット経由で元々維持する。圧縮/暗号化/リパース等の他属性には触れない。ファイルは
  成功パスの元削除前、フォルダは `_apply_dir_time` 内で適用する。
- **移動時、元に残ったファイルがあるフォルダは削除しない**。エラー/無視/先頭スキップで
  残存したファイルを検知し、空フォルダのみ深い階層から削除する（要件）。
- **処理順は決定論的**（`os.walk` の dirs/files を `sort()`）。「先頭N件スキップ」機能が
  再現性に依存するため、この順序を崩さないこと。
- コピー先がコピー元の内部/同一の場合は中止する。

## 無視リスト

- 1行1件のファイルマスク。`fnmatch` で判定。
- 区切り文字(`/`,`\`)を含むパターンは相対パス全体に、含まないパターンはファイル名にマッチ。
- `#` 始まりの行と空行は無視。UTF-8（インポートは BOM 許容）でテキスト入出力。
