# CLAUDE.md

このリポジトリで作業する際のガイダンス。

## プロジェクト概要

**Skipferry** — フォルダのコピー/移動を行う Python (Tkinter) GUI ツール。単一ファイル構成。
名称は skip（破損ファイルを飛ばす）+ ferry（健全なファイルを渡し船で運ぶ）の造語。
**主目的は破損ファイル対策**: アプリやOSを巻き込んで固まるタイプの破損ファイルがあっても、
処理順で先頭 N 件をスキップして健全なファイルだけを運び切ること。
加えてサブフォルダを含む再帰処理、タイムスタンプ維持、ベリファイ、ごみ箱送り、
無視リストなどを備える。

（リポジトリのフォルダ名は歴史的経緯で `putoff-file-move` のまま。ツール名は Skipferry。）

## 実行

```
python skipferry.py
```

- 追加パッケージが無くても起動・動作する（外部ライブラリはすべて任意）。
- ごみ箱送りは `send2trash` があれば使用し、無ければ Windows の `SHFileOperation`(ctypes)
  へ自動フォールバックする。`send2trash` は任意（`pip install Send2Trash`）。
- ドラッグ&ドロップは `tkinterdnd2` があれば有効（`pip install tkinterdnd2`）。無い場合は
  D&D 無効で通常起動する（欄下に導入案内を表示）。
- 主対象は Windows。ごみ箱送りフォールバックは Windows 専用。

## 構成

- `skipferry.py` — 全ロジックとGUIを含む単一ファイル。

## アーキテクチャ

- **国際化 (i18n)** — UI 文言・ログ・エラーメッセージはすべてモジュール先頭の
  `TRANSLATIONS`(`ja`/`en`) と `t(key, lang=None, **kwargs)` を経由する。**新規文言は必ず
  両言語のキーを追加し、リテラルを直接埋め込まないこと**（`t()` は欠落キーを日本語→キーの順で
  フォールバックする）。現在の言語は module-global `CURRENT_LANG`（既定 `ja`）。GUI 上部の
  ラジオボタンで切替え、`App._on_lang_change` が全ウィジェットを破棄して `_build_ui` で
  再構築する（Tk 変数は `_init_vars` で一度だけ生成し保持、無視リスト/ログ Text の内容は
  退避して復元）。**ワーカーは開始時の言語を `cfg["lang"]` で固定**し `self.tr()` で翻訳するため、
  実行中に UI 言語を切替えてもログ言語はぶれない（実行中は言語切替を無効化）。
  モジュール関数 `plan_operation`/`check_dest_path_valid` は `cfg["lang"]`/`lang` 引数で言語を受ける。
- **処理計画 (`plan_operation`, モジュール関数)** — 設定からコピー先ルート・処理対象
  ファイル一覧(無視/先頭スキップ適用後・ソート済)・エラーを算出する。
  **ワーカー本処理と実行前プレビューの両方がこの関数を共有する**（結果がずれないため）。
  コピー先パスの妥当性検査 `check_dest_path_valid`（Windowsで使用不可の文字
  `< > : " | ? *`・制御文字・予約名 CON/PRN/…・末尾の空白/ドット）もここで行い、
  問題があれば `error` に載せる。ドライブの `:` と区切りは splitdrive で除外して構成要素を見る。
- **GUI ドラッグ&ドロップ** — `tkinterdnd2` があれば基底クラスを `TkinterDnD.Tk` に切替え
  （`_BaseTk`）、コピー元/先の Entry に `DND_FILES` を登録。`_on_drop` は `tk.splitlist` で
  波括弧括りのパスを分解し、ファイルが落ちた場合はその親フォルダを採用する。
- **GUI (`App`, tk.Tk)** — パス指定・オプション・無視リスト・進捗・ログを構築。
  実処理はワーカースレッドに委譲し、GUIは触らない。実行時はまず `plan_operation` で
  計画を作り、`_show_preview` で処理先ファイル(先頭10件)を提示して確認を取ってから開始する。
- **ワーカー (`CopyMoveWorker`, threading.Thread)** — 実際のコピー/移動/検証/削除を実行。
  一覧作成は `plan_operation` に委譲。結果は `queue.Queue` へ
  `("log"|"progress"|"ignore_add"|"done", payload)` 形式で送る。
- **ログファイル出力** — `cfg["log_file"]` にパスがあれば `run()` が追記(`a`)モードで開き、
  `self.log()` が GUI へ送るのと同時に時刻付きでファイルへも書く(逐次 flush)。開始/終了に
  区切り行を出す。書き込み失敗は処理本体を止めない。GUI 側は `_resolve_log_file` で
  パスを決定（空欄ならコピー先の隣に `skipferry_log_<日時>.txt` を自動生成）。
  加えて現在のログ欄を任意保存する `_save_log`（実行不要でいつでも可）も持つ。
- **設定内容の書き出し** — `_run` の冒頭で `_log_settings` がオプション設定
  （コピー元/先・サブフォルダ作成・動作/移動時の元処理・ベリファイ・エラースキップ・
  自動無視・先頭スキップ件数・ウェイト・無視パターン数・ログファイル）をログへ出力する。
  無視パターン数はコメント行(`#`)・空行を除外して数える。GUI ログ/ログファイルの両方に出る。
- **ファイル処理ログの前後** — 各ファイルはコピー**直前**に `[done/total] <rel>`(`log_item`)、
  成功確定**後**（コピー→検証→移動時の元削除が全て通った後）に `  [完了] <rel>`(`log_item_ok`)
  を出す。**処理前ログは維持すること**（固まった場合にログ最終行が原因ファイルを指し、無視/
  先頭スキップ対象を特定できるという主目的に直結するため）。エラー時は `  [エラー] ...` を追加。
- **GUI ↔ ワーカー連携** — `App._poll_queue` が `after(100ms)` でキューをポーリングし
  ログ・進捗バー・無視リスト追加を反映。停止は `worker.stop_flag`(Event)、一時停止は
  `worker.pause_flag`(Event) で協調的に行う。
  **ワーカースレッドから Tkinter ウィジェットを直接操作しないこと**（キュー経由必須）。
- **一時停止/再開・ウェイト** — いずれもワーカー内でフラグを見て協調的に待機する。
  一時停止は `pause_flag`。ワーカーは各ファイル処理**前**に `_wait_if_paused` を呼び、
  一時停止中は 0.1 秒刻みで待機する（停止要求が来れば即座に抜ける）。GUI 側の
  「一時停止」ボタンがラベルを「一時停止 ⇄ 再開」でトグルする。**一時停止中でも停止できる
  よう `_stop` は `pause_flag` を解除してから `stop_flag` を立てること**。
  OS/アプリ負荷軽減のウェイトは `cfg["wait_ms"]`(ミリ秒)。各ファイル処理**後**に
  `_throttle_sleep` が挟む。待機は 0.1 秒刻みで停止/一時停止に応答する（長いウェイト中でも
  中断・一時停止が効くようにするため、まとめて `sleep` しないこと）。

## 設計上の重要な決定（変更時は維持すること）

- **移動は「コピー→検証→元削除」で統一** している（`os.rename` は使わない）。
  これにより別ドライブ間移動とベリファイを両立させている。
- **タイムスタンプ維持**: ファイルは `shutil.copy2`。フォルダは全ファイル処理**後**に
  `os.utime` を適用する（先に設定すると子ファイル書き込みで親mtimeが変わるため）。
- **ファイル属性/ACL/副次ストリーム(ADS)は操作しない**。システムデフォルトに従う（要件）。
- **移動時、元に残ったファイルがあるフォルダは削除しない**。エラー/無視/先頭スキップで
  残存したファイルを検知し、空フォルダのみ深い階層から削除する（要件）。
- **処理順は決定論的**（`os.walk` の dirs/files を `sort()`）。「先頭N件スキップ」機能が
  再現性に依存するため、この順序を崩さないこと。
- コピー先がコピー元の内部/同一の場合は中止する。

## 無視リスト

- 1行1件のファイルマスク。`fnmatch` で判定。
- 区切り文字(`/`,`\`)を含むパターンは相対パス全体に、含まないパターンはファイル名にマッチ。
- `#` 始まりの行と空行は無視。UTF-8（インポートは BOM 許容）でテキスト入出力。

## テスト

自動テストのファイルは無い。変更後は以下を手動確認する:

- 構文チェック: `python -c "import py_compile; py_compile.compile('skipferry.py', doraise=True)"`
- コアロジックは GUI を起動せず `CopyMoveWorker` を直接インスタンス化し、
  `queue.Queue()` を渡して `.run()` を呼べばヘッドレスで検証できる
  （コピー/移動/検証/無視/先頭スキップ/フォルダ削除/タイムスタンプ維持）。
